
<contexte>
Il s'agit de la fourniture d'une valeur pour l'information suivante :
Nom du champ : '{field_name}'.
Description du champ : '{field_desc}'.
</contexte>
<contexte>
Voici la liste des contraintes que doit respecter ce champ :
{field_infos}
</contexte>
<contexte>
L'utilisateur a précédemment fourni la valeur suivante pour ce champ : '{field_previous_value}'.
Mais cette valeur n'est pas valide pour la ou les raisons suivantes : '{field_previous_value_error_message}'
</contexte>

<objectifs>
Procède étape par étape :
1- Analyse les raisons pour lesquelles la valeur précédemment fournie n'est pas valide.
2- Explique à l'utilisateur pourquoi la valeur qu'il avait fournie n'est pas valide.
3- Décris les contraintes auxquelles doit répondre la valeur du champ, dans un français simple et compréhensible, y compris pour la règle regex spécifiée, e cas échéent.
4- Puis demande à l'utilisateur de fournir une valeur valide pour le champ spécifié.
</objectifs>
