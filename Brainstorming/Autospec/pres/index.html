01 — GPT-3.5 · les débuts

Le début du code par IA

Novembre 2022, GPT-3.5 ; quelques mois plus tard, GPT-4. L'IA générative entre dans le quotidien du développeur.

/ le chatbot

Demander & copier-coller

On décrit un besoin dans l'interface ChatGPT, on récupère un bout de code — puis on le copie-colle dans son IDE.

/ dans l'IDE

Copilot

L'autocomplétion ligne à ligne, puis une première fenêtre de chat intégrée : transformer une demande textuelle en code, sans quitter l'éditeur.

02 — Les agents de codage

L'agent prend le clavier

L'agent s'intègre à l'IDE avec Copilot et Cursor, puis s'émancipe dans le terminal avec Claude Code, et jusqu'au cloud avec Codex.
Il propose une approche décomposée en deux temps :

/ étape 1

Planifier

Réfléchir, spécifier, cadrer ce que l'on va faire — avant d'écrire la moindre ligne.

/ étape 2

Implémenter

Écrire le code, itérer, corriger — l'exécution guidée par le plan.

06
06 — Le futur

La software factory

Et si la chaîne de production logicielle tournait d'elle-même ? L'humain n'intervient plus que par incursions limitées, de haut niveau et adaptatives. C'est l'initiative Autospec.

/ limité

Incursions ciblées

La chaîne avance seule ; l'humain n'est sollicité qu'aux points de décision qui comptent vraiment.

/ haut niveau

Décider, pas exécuter

On arbitre l'intention et les choix structurants — pas les détails d'implémentation.

/ adaptatif

Boucle qui s'ajuste

Le niveau et le moment du contrôle humain s'adaptent à la criticité et à la confiance.

→ Automatiser la chaîne de production — et garder l'humain là où il crée le plus de valeur.

07
07 — Tout part d'un prompt

Projet Autospec

Tout commence par un simple prompt en langage naturel :

/goalcréer une usine logicielle agentique qui transforme un simple brief en langage naturel en logiciel testé, déployé, réellement fonctionnel.
/ méthode

L'intelligence dans la boucle

Des agents à rôles (PM · PO · QA · Dev, issus de BMAD) déroulent un cycle BDD puis TDD strict — critères Gherkin d'abord, tests ensuite, code enfin — porté par une boucle ReAct coder → vérifier → corriger, jusqu'aux tests e2e Playwright qui valident le résultat réel.

/ principes

Qualité par conception

Les principes de conception sont câblés dans la boucle : architecture SOLID & séparation des responsabilités, et des agents qui cherchent l'existant avant de créer pour rester DRY. Structurel, pas optionnel.

/ ambition

Vérifiabilité déterministe

Un résultat réellement vérifié à chaque étape — le socle de boucles de feedback rapides, où la confiance ne repose plus sur la chance.

Une équipe d'agents préconfigurés

Des rôles issus de BMAD Method, enchaînés du cadrage jusqu'au code testé.

Rédige le brief et cadre le besoin : le quoi et le pourquoi.

→
PO · Product Owner

Découper

Structure et priorise en EPIC & US, écrit les critères et tests d'acceptance.

→
DEV · Test-Driven

Construire

Contrats BDD d'abord, puis full TDD : tests → code → exécution, en boucles de rétroaction.

→←
QA · Quality Assurance

Valider

Tests automatisés de bout en bout : scénarios e2e pilotés par Playwright, jusqu'au feu vert.

→ L'intelligence organisationnelle vit dans la boucle agentique, pas dans la tête de l'opérateur.
