01
01 — GPT-3.5 · les débuts

Le début du code par IA

Novembre 2022, GPT-3.5 ; quelques mois plus tard, GPT-4. L'IA générative entre dans le quotidien du développeur.

/ le chatbot

Demander & copier-coller

On décrit un besoin dans l'interface ChatGPT, on récupère un bout de code — puis on le copie-colle dans son IDE.

/ dans l'IDE

Copilot

L'autocomplétion ligne à ligne, puis une première fenêtre de chat intégrée : transformer une demande textuelle en code, sans quitter l'éditeur.

02
02 — Les agents de codage

L'agent prend le clavier

L'agent s'intègre à l'IDE avec Copilot et Cursor, puis s'émancipe dans le terminal avec Claude Code, puis et le cloud avec Codex.
Il propose une approche décomposée en deux temps :

/ étape 1

Planifier

Réfléchir, spécifier, cadrer ce que l'on va faire — avant d'écrire la moindre ligne.

/ étape 2

Implémenter

Écrire le code, itérer, corriger — l'exécution guidée par le plan.

03
03 — Plus d'autonomie

Les Harnesses
des agents plus autonomes

— exemple : Claude Code.
L'agent ne fait pas que coder :
Boucle ReAct le pousse à agir, vérifier et corriger — linter, compilation, tests unitaires et e2e.

↳ déléguer

Sub-agents

Des sous-agents spécialisés, lancés à la demande pour des tâches ciblées.

↳ coordonner

Agent teams

Des équipes d'agents qui se partagent le travail et se synchronisent.

↳ orchestrer

Dynamic workflows

Des flux qui s'adaptent en temps réel : fan-out, vérification, synthèse.

↳ outiller

Skills

Des compétences décorrélées des agents, mobilisables à la demande.

04
04 — Le changement de paradigme

Spec Driven Development

Le grand retour du cycle en V : coder et tester de moins en moins, spécifier de plus en plus. On définit un but — l'agent décide du comment.

◆BMAD Method ◆SpecKit ◆& au-delà…
/loop
Boucler jusqu'à ce que les conditions soient atteintes.
/goal
Continuer jusqu'à ce que l'objectif soit atteint.

Vers des agents proactifs et adaptatifs

exemples — OpenClaw · Hermes Agent
/ orchestration

Choisir ses agents

Définir, de façon transparente, quels agents mobiliser pour répondre au besoin.

06
06 — Créer son propre harness

Projet Autospec

Construire soi-même un harness pour produire du logiciel à partir de spécifications. Tout ce qu'on a vu — mais qu'on bâtit nous-mêmes.

/ principe

Méthodologie top-bottom

Injecter une démarche descendante : du besoin vers le code, étape par étape.

/ ambition

Intelligence organisationnelle

Encoder le maximum de méthode et de rôles directement dans les boucles agentiques.

06
06 — Projet AutoSpec · l'équipe

Une équipe d'agents préconfigurés

Des rôles issus de BMAD Method, enchaînés du cadrage jusqu'au code testé.

PM · Product Manager

Définir

Rédige le brief et cadre le besoin : le quoi et le pourquoi.

PO · Product Owner

Découper

Structure et priorise en EPIC & US, écrit les critères et tests d'acceptance.

DEV · Test-Driven

Construire

Contrats BDD d'abord, puis full TDD : tests → code → exécution, en boucles de rétroaction.

→ L'intelligence organisationnelle vit dans la boucle agentique, pas dans la tête de l'opérateur.
