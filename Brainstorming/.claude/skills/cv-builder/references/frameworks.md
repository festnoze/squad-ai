# Bullet-Point Frameworks

## Google XYZ Formula (Default)

Template: `Accomplished [X] as measured by [Y] by doing [Z]`

- **X** = Accomplishment (starts with action verb)
- **Y** = Metric or measurement (always a number)
- **Z** = Method, tools, or approach

### Examples

```
Reduced API response latency by 62% (from 850ms to 320ms p95) by redesigning
the caching layer with Redis cluster, eliminating 3 recurring production incidents/month.

Accelerated model training throughput by 4x by migrating the pipeline from single-GPU
to distributed training on an 8-node Kubernetes cluster.

Increased recommendation click-through rate from 3.2% to 5.8% (+81%) by implementing
a collaborative filtering model with PyTorch, generating ~$1.2M in additional annual revenue.
```

## STAR Method (Situation, Task, Action, Result)

Use for complex, multi-stakeholder achievements that need context.

On a resume, compress Situation+Task into a brief clause. Action and Result carry the weight.

### Example

```
Weak:  Responsible for improving system performance.
STAR:  Reduced API response latency by 62% (from 850ms to 320ms p95) by redesigning
       the caching layer with Redis cluster, eliminating 3 recurring production incidents/month.
```

Reserve STAR for 2-3 highest-impact achievements per role. Simpler bullets use XYZ.

## CAR Method (Challenge, Action, Result)

More concise than STAR -- eliminates the separate Task component. Best for technical problem-solving.

### Example

```
Resolved critical memory leak in production microservice by profiling heap allocations
and refactoring connection pooling logic, reducing crash incidents from 12/month to zero
and saving $45K in annual downtime costs.
```

## PAR Method (Problem, Action, Result)

Synonymous with CAR. Use whichever label resonates.

## Decision Guide

| Framework | Best For | Length |
|-----------|----------|--------|
| **XYZ** | Metric-heavy achievements, FAANG applications, default choice | Medium |
| **STAR** | Complex multi-stakeholder achievements needing context | Longest |
| **CAR/PAR** | Technical problem-solving, debugging, optimization stories | Shortest |

## Quantification Categories

| Category | Examples |
|----------|----------|
| Financial | Revenue generated, costs saved, budget managed |
| Scale | Users served, requests/sec, data volume processed |
| Efficiency | Time saved, speed improvement, automation rate |
| Quality | Error rate reduction, accuracy improvement, uptime % |
| Team/Scope | Team size, stakeholders, markets served |
| ML-Specific | F1/AUC improvement, p95 latency, inference cost, RMSE reduction |

### When Exact Numbers Are Unavailable

- Use ranges: "Managed 50-75 client accounts"
- Use frequency: "Deployed 3 production models per quarter"
- Use scope: "Across 4 geographic regions"
- Estimate conservatively: approximate percentages are still powerful
